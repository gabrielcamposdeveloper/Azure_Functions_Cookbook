# 📨 Azure Queue Storage Pipeline

In this Cookbook, we explore how to build a decoupled, asynchronous messaging pipeline using **Azure Queue Storage** and the **.NET 8 Isolated Worker Model**. 

This guide explains how to produce messages (via an HTTP Output Binding or a standalone Console App) and consume them automatically using a Queue Trigger.

---

## 🧠 How It Works (The Concept)

Messaging queues are perfect for decoupling systems and handling background processing. Our pipeline demonstrates two ways to send a message to a queue, and one way to process it:

1. **Entry Point 1 (The Uploader):** A standalone C# Console App (`QueueUploader`) connects directly to the Azure Storage SDK and pushes a Base64-encoded message into the queue.
2. **Entry Point 2 (Output Binding):** An HTTP-triggered Azure Function receives a POST request. Instead of using the SDK manually, it uses an **Output Binding** to seamlessly drop the request body into the queue.
3. **The Consumer (Queue Trigger):** A background Azure Function constantly monitors the `orders` queue. As soon as a message arrives (from either entry point), it wakes up, reads the message, and processes it.

---

## ⚙️ Technical Deep Dive

Here is a quick look at the technical details inside our code:

### 1. `QueueUploader` (The External Producer)
When using the native Azure SDK (`Azure.Storage.Queues`) to send messages to a queue that an Azure Function will consume, **you must encode the message in Base64**. Azure Functions naturally expect queue messages to be Base64-encoded to prevent data corruption.
Our uploader handles this automatically: `Convert.ToBase64String(Encoding.UTF8.GetBytes(message))`.

### 2. `QueueOutputBindingExample` (The HTTP Producer)
Because an HTTP function needs to return an HTTP response *and* send a message to the queue, we use the **Multiple Output** pattern. 
We return a custom class (`QueueOutputResponse`) decorated with `[QueueOutput("orders")]` for the message and `[HttpResult]` for the HTTP response. Azure handles the routing behind the scenes!

### 3. `QueueTriggerExample` (The Consumer)
This function uses the `[QueueTrigger("orders")]` attribute. The Functions Host automatically handles polling the queue, fetching the message, converting it from Base64 back to plain text, and delivering it to the `message` parameter.

## 🚀 Step-by-Step: How to Run the Pipeline

Follow these steps to see the asynchronous messaging cycle in action on your local machine.

### Prerequisites
Make sure your **Azurite** emulator is running (Queue Storage relies on port 10001).

### Step 1: Start the Azure Functions Host
Open a terminal in the root of your `AzureFunctionsCookbook` project and run:
```bash
func start

```

Wait until the ASCII logo appears and the `queue-output` HTTP route is listed. **Leave this terminal open.**

### Step 2: Inject a Message via the Uploader

Open a **second terminal**, navigate to the `QueueUploader` console app folder, and run it:

```bash
cd QueueUploader
dotnet run

```

*What happens:* The console app connects to your local Azurite, ensures the `orders` queue exists, and uploads the message *"New order generated by the QueueUploader!"*.

### Step 3: Inject a Message via HTTP

You can also push messages using the Output Binding. Using `curl` (or Postman/Thunder Client), run this command in your terminal:

```bash
curl -X POST http://localhost:7071/api/queue-output -H "Content-Type: text/plain" -d "Process this priority order via HTTP!"

```

*What happens:* The Function receives your POST request, returns a success message to your terminal, and automatically routes your text into the `orders` queue.

### Step 4: Watch the Trigger React

Go back to your **first terminal** (where `func start` is running). Because you sent two messages, you should see the trigger wake up twice and log the contents:

```text
info: AzureFunctionsCookbook.QueueTriggerExample[0]
      ==============================================
info: AzureFunctionsCookbook.QueueTriggerExample[0]
       🚀 [NEW MESSAGE DETECTED IN QUEUE] 🚀
info: AzureFunctionsCookbook.QueueTriggerExample[0]
       📦 Message Content: New order generated by the QueueUploader!
info: AzureFunctionsCookbook.QueueTriggerExample[0]
      ==============================================

```

🎉 **Congratulations!** You just executed a complete, event-driven messaging pipeline locally.
